1. Executive Summary 
A lightweight, mobile-first Progressive Web App designed for Indian village retail stores. 
Customers can browse products, place pre-orders, track orders, maintain khata (credit), and 
reorder frequently purchased groceries before visiting the store. 
The app solves real operational problems for a local dukaan owner: chaotic walk-in traffic, 
repeated verbal orders, manual khata tracking, uncertain customer arrivals, and lack of order 
history. 
  
Optimized for: 
• Simplicity and low-data usage 
• Repeat grocery ordering with Hinglish UI 
• Fast pickup workflow for village customers 
• Zero recurring cost — runs entirely on Firebase free tier 
• Fully manageable by a single owner using only a smartphone 
2. Product Vision 
"A Smart Digital Dukaan Assistant for Village Retail" 
  
This product is NOT intended to become a grocery marketplace, a delivery platform, or a 
multi-store ecommerce application. 
  
Focused on: 
• Faster order preparation and repeat customer convenience 
• Simplified digital khata management 
• Customer trust and loyalty 
• Reduced manual effort for the owner 
• Digital ordering habit formation in village customers 
3. Problem Statement 
Current Situation 
Customers currently visit the store without knowing if products are available, whether the 
order can be packed quickly, or how long pickup may take. 
  
The owner manually listens to verbal orders, packs items while customers wait, maintains 
khata records in notebooks, and tracks regular customer preferences mentally. 
  
This creates customer wait times, operational confusion, missed sales, repeated manual 
work, and no digital record of demand patterns.

9. Authentication & Security 
[UPDATED]  Authentication 
UPDATED: Switched to mobile number as primary identifier (matching Indian app norms — 
GPay, Meesho, PhonePe). Username/password removed. WhatsApp OTP used instead of SMS 
to eliminate all OTP costs. 
  
Registration Flow 
Required fields: 
• Full Name 
• Mobile Number (primary identifier) 
• WhatsApp OTP verification (only during initial signup) 
  
No username or password required. Mobile number is the login identity. This matches what 
village users already do on WhatsApp, Paytm, and GPay. 
  
Login Flow 
• Enter mobile number → receive WhatsApp OTP → logged in 
• Session persists for 30 days (no repeated OTP on same device) 
• OTP only required on new device or session expiry 
  
OTP Channel — WhatsApp (Free) 
OTP is sent via WhatsApp message, not SMS. WhatsApp Business API free tier supports 
this at zero cost. Recommended provider: wati.io or Twilio WhatsApp Sandbox (both have 
free tiers for low volume). 
  
Why WhatsApp OTP: Village users already have WhatsApp on their phone. A WhatsApp 
message is more visible than an SMS, costs nothing, and does not require a paid SMS gateway. 
  
Security Requirements 
• Session persistence — stay logged in for 30 days on same device 
• Rate limiting via Firebase Auth built-in protections (no custom rate limiting needed for 
100 users) 
• Firestore Security Rules — users can only read/write their own data 
• Owner account — separate role flag in Firestore user document 
  
Rule Limit 
OTP requests Max 5/hour (Firebase Auth built-in) 
Session duration 30 days before re-authentication 
Owner account Single owner UID — hardcoded in security rules 
  
10. Language Strategy 
[UPDATED]  Language 
UPDATED: Language toggle removed for MVP. Single Hinglish mode only. This halves 
translation work and village users will not meaningfully use a Hindi/Hinglish toggle. Toggle can be 
added in Phase 2 based on actual feedback. 
  
Application Language: Hinglish 
All UI copy is in Hinglish — a natural mix of Hindi and English that village customers use 
daily. Examples: "Order karo", "Abhi available hai", "Mera khata", "Dobara order karo". 
  
Product Naming Model 
Each product stores multiple name variants to support search across languages: 
• Hinglish display name: "Sarson ka Tel" 
• Hindi name: "ससससस सस ससस" 
• English aliases: "Mustard Oil", "mustard" 
• Search keywords: "tel", "sarson", "mustard" 
  
Search Requirements 
Search must support Hindi input, Hinglish input, English input, and spelling variations. 
Example: daal / dal / ससस — all should return the correct product. 
  
Order Notes 
Order notes accept any input — Hinglish, Hindi Devanagari, or English. The keyboard is 
whatever the customer's Android default is. Example note: "Kam mirch wala dalna" or "सस 
ससससस सससस ससससस". 
Removed: The PRD v3 example note was in pure Devanagari which requires keyboard 
switching. Hinglish roman input is the natural behavior for this audience.

[UPDATED]  Product Variants — Critical Addition 
NEW: Product variants are required for grocery catalogs. Without variants, the owner must create 
separate catalog entries for Atta 1kg, Atta 2kg, Atta 5kg — bloating the catalog and breaking 
search. Variants solve this at the data model level. 
  
Variant Data Structure 
Each product has an optional variants array. If variants exist, the customer selects a variant 
after tapping the product. If no variants, quantity is entered directly. 
  
Example product structure: 
{ "name": "Atta", "variants": [ { "label": "1 kg", "price": 45 }, { "label": "2 
kg", "price": 88 }, { "label": "5 kg", "price": 210 } ] } 
  
Availability States 
The application does NOT show exact inventory quantities. Instead: 
• Available Today — normal ordering enabled 
• Running Low — order enabled with caution message: "Jaldi karo, sirf thoda bacha 
hai" 
• Out of Stock — ordering disabled for this product 
This matches real village grocery behavior and reduces inventory maintenance complexity.

14. Cart System 
Cart Features 
• Add / remove quantity per item 
• Variant selection per item (e.g., 2kg vs 5kg Atta) 
• Cart persistence — saved to Firestore, survives app close 
• Line item totals and grand total 
• Order notes field (accepts Hinglish / Hindi / English text) 
  
[UPDATED]  Minimum Order Value 
NEW: A minimum order value prevents trivial orders that waste owner packing time. 
• Default minimum order: Rs. 50 
• Owner can configure this from the dashboard (range: Rs. 20 to Rs. 500) 
• Cart shows "Minimum order Rs. 50" when total is below threshold 
• Order placement is disabled until minimum is met 
  
Price Snapshot at Order Placement 
• Item price is captured (snapshotted) at the moment of order placement 
• If owner updates prices in the morning before opening, all subsequent orders use the 
new price 
• Cart shows a "Price updated, please review" banner if price changes while item is in 
cart 
Note: Owner updates prices once in the morning — this is the agreed workflow. No complex 
price-change conflict resolution needed.

[NEW]  16. Store Open / Closed Toggle 
NEW: This is a critical missing feature from v3.0. Without it, customers can place orders at

Owner Control

• Toggling closed immediately hides the order placement button for all customers 
• Owner can set a reopen message: "Kal subah 9 baje kholenge" 
  
Customer Experience 
• When store is closed, catalog is still browsable (offline-cache works) 
• Order button is replaced with: "Dukaan abhi band hai — [owner message]" 
• 5–10 minute pickup slot is hidden when store is closed 
  
Auto-Close (Optional) 
• Owner can set operating hours (e.g., 8 AM – 8 PM) 
• App automatically shows closed state outside those hours 
• Owner can still manually override to open/close at any time

18. Partial Order Handling 
[UPDATED]  Partial Order Handling 
UPDATED: v3.0 did not define behavior when ALL items in an order become unavailable. This is 
now specified. 
  
Some Items Unavailable 
• Owner removes unavailable items from the order 
• Remaining order continues normally 
• Total updates automatically 
• Customer receives push notification: "Aapke order mein se [item] nahi mila — baki 
order tayar ho raha hai" 
  
ALL Items Unavailable (New) 
• If owner removes all items, the order is auto-cancelled 
• Customer receives notification: "Aaj aapka order available nahi hai — sorry!" 
• Khata is NOT affected — no amount is added for a fully cancelled order 
• Trust score is NOT reduced — this is a store-side cancellation, not customer fault

22. Khata (Credit) System 
[UPDATED]  Khata — Two Additions 
UPDATED: Two new features added: (1) Per-customer credit limit to prevent unchecked debt 
accumulation. (2) Customer dispute flag so customers can report incorrect entries. 
  
Khata Features 
• Running balance per customer 
• Full ledger history with date, type, amount, note 
• Partial payments and payment notes 
• Settlement tracking 
• Manual adjustment support by owner 
  
Credit Limit (New) 
• Each customer has a configurable credit limit (default: Rs. 500) 
• Owner can change the limit per customer from the dashboard 
• When a customer's outstanding khata balance reaches their limit, new orders using 
khata are blocked 
• Customer sees: "Aapka khata Rs. 500 ho gaya — pehle thoda pay karo" 
• Owner sees an alert on the order card: "Khata limit pe hai" 
  
Dispute Flag (New) 
• Customer can tap "Galat lag raha hai" on any khata entry 
• This sends the owner a notification: "[Customer name] ne entry flag ki hai — check 
karo" 
• No complex dispute resolution flow — owner resolves manually and can 
update/delete entries 
• Builds customer trust by giving them a voice on their own account

[NEW]  25. Store Discovery & Customer Onboarding 
NEW: v3.0 had no mechanism for how customers find or install the PWA. This is a critical gap — 
without discoverability, adoption never happens. 
  
Store QR Code 
• Owner dashboard generates a unique store QR code 
• Owner prints and sticks the QR code at the store counter 
• Scanning the QR code opens the PWA directly in the customer's browser 
• First-time visitors see the registration screen; returning users are auto-logged in 
  
WhatsApp Share Link 
• Owner can copy a shareable link from the dashboard 
• Owner sends this link to customers on WhatsApp: "Hamare dukaan ki app download 
karo, order easy ho jayega!" 
• Link opens the PWA on mobile — no App Store needed 
  
PWA Install Prompt 
• First visit shows: "Hone Dukaan apne phone mein add karo" with Add to Home 
Screen prompt 
• This is a standard PWA feature — no app store submission needed 
[NEW]  26. Store Info Page 
NEW: Customers need a way to find store address, phone, and hours from within the app. 
  
• Store name and owner name 
• Address (text field) 
• Phone number (tappable — opens dialer) 
• Operating hours 
• "Raste mein" button — opens Google Maps with store address 
Owner configures store info during initial setup. Editable from dashboard settings. 
  
[NEW]  27. Owner-to-Customer Order Notes 
NEW: Owner needs a way to communicate substitutions or questions without calling the 
customer. Example: 'Sarson tel nahi hai — refined chalega?' 
  
• Owner can add a note to any order in the dashboard 
• Customer receives a push notification and sees the note on their order status screen 
• Customer can respond with one of two taps: Theek Hai (OK) or Nahi Chahiye 
(Cancel this item) 
• Simple binary response — no free-text chat to keep it simple
